# 青苗守护者 · 指标与算法说明

本文档汇总后端与 AI 服务中与**指标、统计、评分、聚合**相关的计算方法及数据来源。路径相对于仓库根目录 `qingmiao-guardian`。

---

## 1. 数据大屏 `backend/app/services/dashboard_service.py`

| 指标 | 计算方法 |
|------|----------|
| 学校 / 年级 / 班级 / 学生 / 教师数 | 各实体表 `is_deleted=False` 的 `count()` |
| 今日预警数 | `Alert`，`date(created_at) == 当天` |
| 未结预警数 | `status in ('open','acknowledged')` 的条数 |
| 近 7 天视频任务数 | `AnalysisTask`，`date(created_at) >= 当天-7` |
| 近 7 天量表测评数 | `ScaleAssessment`，同上 |
| **心理健康指数 `psy_index`** | 近 30 天所有 `EmotionTimeline.score` 的**算术平均**，`round(..., 1)`；无数据时 **85.0** |
| 预警等级分布 | `Alert.level` 分组 `count` |
| **班级活力 `engagement`** | 每班：该班学生近 30 天 `EmotionTimeline.score` 的**算术平均**，无数据 **80.0**；按值降序取 Top N |
| **近 30 天情绪曲线** | 按日 `group_by(record_date)`，`avg(score)`，再 `round` |
| **今日行为分布** | 今日 `BehaviorRecord` 按标签分组计数后，经 `_normalize_behavior_cn`（见 `video_service`）归并到**标准八类**；全无数据时使用内置演示柱状数据 |
| **今日情绪分布** | 今日 `EmotionRecord.emotion_cn` 分组计数；无数据时使用演示数据 |
| 近期预警流 | 取最新 N 条，`score` 等为库字段 |
| **高风险学生榜** | `status in ('open','acknowledged')`，按 **`Alert.score` 降序** |

## 2. 课堂视频与任务报告 `backend/app/services/video_service.py` + `backend/app/tasks/video_worker.py`

### 2.1 元数据

- **文件 MD5**：上传内容 `hashlib.md5`。
- **时长 / fps / 宽高**：OpenCV `VideoCapture`；`duration = frame_count / fps`，失败则默认 `60s, fps=0`。
- **size_mb**：`size_bytes / 1024 / 1024`，保留 2 位小数。

### 2.2 行为归一 `_normalize_behavior_cn`

将历史/英文 `label` 映射到固定**八类中文**（`BEHAVIOR_EIGHT_CN`），用于统计与报告展示一致；规则为条件映射表，无法匹配时默认为 **「转头」**。

### 2.3 任务报告 `get_task_report`

- **时间轴 bucket**：`bucket = 5` 秒，`b = round(frame_time / 5) * 5`，行为与情绪在该时间桶内计数。
- **八类行为计数 `behavior_by_class`**：每条记录经归一后累加。
- **派生量**（用于 `metrics`）  
  - `listen` = 「抬头听课」次数  
  - `hand_up` = 「举手」  
  - `head_down` = 「低头写字」+「低头看书」  
  - `discuss` = 「小组讨论」  
- **`engagement_score`（任务报告用）**

  ```text
  engagement = min(100, max(40, int(55 + listen + hand_up*2 + discuss - head_down*2)))
  ```

- **`total_person_detections`**：`listen + 「站立」` 次数。
- **`phone_count`**：当前实现赋值为 **`head_down`**（与字面「手机」不完全一致，以代码为准）。

### 2.4 异步任务汇总 `video_worker` 写入 `task.summary`

- **进度**：`progress = int((已处理帧 + 1) / 总帧 * 100)`。
- **behavior_summary / emotion_summary**：每帧检测结果按标签/情绪累加。
- **`dominant_emotion`**：情绪计数中出现次数最多的类别。
- **`top_behaviors`**：行为计数按 value 降序取前 5。

## 3. AI 实时课堂流水线 `ai_service/pipelines/classroom_pipeline.py`（单帧）

在**当前帧检出人数 `total`** 上计算比例（与后端 `get_task_report` 的参与度**不是同一公式**）：

| 分量 | 权重 | 说明 |
|------|------|------|
| `head_up_rate` | ×40 | `head_up / total` |
| `hand_up_rate` | ×20 | `min(1, hand_up / max(1, total*0.2))`（约 20% 学生举手视为该项满分） |
| `positive_emotion` | ×20 | `emotion_cn in ('高兴','专注','中性')` 的比例 |
| `(1 - lying_rate)` | ×10 | 非趴桌比例 |
| `(1 - phone_rate)` | ×10 | 非使用手机判定比例 |

**`engagement_score` = 上式之和**，`round` 保留 1 位小数；无检出人为 **0**。

**使用手机判定**：手机 bbox 中心落在人体框内，且纵坐标位于人体**下半身**区域。

## 5. 心理健康 `backend/app/services/psychology_service.py`

### 5.1 量表提交 `submit_assessment`

- **每题得分**：取选项 `score`；若题目 **`reverse`**：`score = max_score - raw`。
- **`total_score`**：各题得分之和。
- **维度分 `dim_totals`**：有 `dimension` 的题按维度累加。
- **等级**：在量表 `interpretation` JSON 的区间 `[min, max]` 内匹配 `level / color / advice`。
- **写入情绪时序的 `health_score`**

  - `max_possible` = 各题选项 `score` 最大值的和  
  - `health_score = max(0, 100 - int(total / max(1, max_possible) * 100))`  
  - 含义：**量表原始分越高表示越不健康，转成 0–100 的「健康指数」**。

- **极性 / 风险写入时序**：由量表结果 `color` 映射 `polarity`、`risk_level`。

### 5.2 文本分析 `analyze_text`

- 极性、风险等来自 **AI 客户端**返回。
- **写入时序分数**：`score_map = {正面:85, 中性:75, 负面:55}`，`risk_penalty = {none:0, low:5, medium:15, high:30}`  
  **`hs = max(20, score_map[polarity] - risk_penalty[risk_level])`**

### 5.3 AI 对话 `post_message`

- 对话级 **`risk_level`**：与历史比较取序更高的：`none < low < medium < high`。
- 若本句 **`risk == 'high'`**：写时序 **固定 `score = 30.0`**，极性「负面」，风险 `high`。

### 5.4 时序合并 `_upsert_emotion_timeline`

- 同日同 `source` 已存在：**若新 `score` 更低则覆盖**（保留更差结果）。

### 5.5 学生心理档案 `student_psychology_profile`

- **`latest_score`**：时序最后一条的 `score`，无时序 **80.0**。
- **`top_risk`**：时序各条 `risk_level` 按序取**最高档**。

## 6. 关联分析与综合预警 `backend/app/services/correlation_service.py`

### 6.1 `collect_student_features`

| 特征 | 规则 |
|------|------|
| `psy_score_avg_30d` / `7d` | 近 30 / 7 天 `EmotionTimeline.score` 的 **mean**，无数据 **80.0** |
| `psy_score_min_30d` | 近 30 天最小值，无数据 **80.0** |
| `score_avg` | 最近最多 **10** 条 `Score.score` 的 **mean**，无 **80.0** |
| **`score_trend`** | 成绩列表前半段 vs 后半段均值差：`diff = mean(近半) - mean(远半)`，`>5` 为上升，`<-5` 为下降，否则稳定 |
| `classroom_phone_count` | 最近 500 条行为中 `label in ('cell phone','using_phone')` 的条数 |
| `classroom_sleep_count` | `label in ('sleeping','lying')` |
| `conversation_risk` | 该生所有对话中的 **最高** `risk_level` |

### 6.2 `compute_risk_score`（0–100，越高越危险）

分项**加分**后 `min(100, score)`，再映射等级：

- 心理 30d 均值 `<50` +35；`<65` +22；`<75` +10  
- 量表 `color`：`green:0, blue:5, orange:12, red:18, purple:20, gray:0`  
- 文本风险：`none:0, low:5, medium:10, high:15`  
- 对话风险：`none:0, low:4, medium:9, high:15`  
- 学业：均值 `<60` +8；趋势「下降」+4  
- 课堂：`phone_cnt > 5` 或 `sleep_cnt > 5` 时 +5  

**等级**：`>=60` → `red`；`>=40` → `orange`；`>=20` → `yellow`；否则 `green`。

## 7. 情绪预测 `backend/app/services/forecast_service.py`

- **`_holt_double_exponential`**：Holt 双指数平滑，`α=0.5, β=0.3`，递推 level 与 trend，外推 `horizon` 步。
- **预测值**：逐点限制在 **[0, 100]**，保留 1 位小数。
- **`avg_recent_7d`**：历史序列**最后至多 7 点**的均值，无序列 **80.0**。
- **`avg_forecast_7d`**：预测 7 点均值。
- **`trend_diff`**：`avg_forecast_7d - avg_recent_7d`，据此生成趋势文案与颜色（阈值 ±2、±5）。
- **`risk_alert`**：预测序列中是否存在 **`<50`**。

## 8. 群体聚类 `backend/app/services/cluster_service.py`

- **特征向量**（每生）：`[psy_30d, score_avg, scale_score, phone_count + sleep_count]`。
- **归一化**：每维 min-max 到 [0, 1]，极差为 0 时填 **0.5**。
- **K-Means**：手写实现，`k=5`，`random.seed(2026)`，最多 30 轮迭代。
- **2D 展示坐标**：取**前两维**（心理健康指数、学业平均分）为 x、y。
- **簇画像**：按簇内 `avg_psy / avg_score / avg_scale / avg_abnormal` 与阈值规则匹配 `CLUSTER_PROFILES` 中的类型名称与建议。

## 9. 干预闭环 `backend/app/services/intervention_service.py`

- **`success_rate`**：`status in ('resolved','closed')` 的数量 / 全部预警数 × 100%。
- **`avg_hours`**：存在 `resolved_at` 的预警，`(resolved_at - created_at)` 转为小时后的**算术平均**。
- **单预警 `alert_journey`**：以**首次干预记录日期**为轴（无干预则用 `acknowledged_at` 或当天），取轴前 **14 天**与轴起 **14 天内**的 `EmotionTimeline.score` 分别求平均 → `avg_before`、`avg_after`，**`improvement = avg_after - avg_before`**。

## 11. 知识图谱 `backend/app/services/graph_service.py`

- 节点 **`symbolSize`**：学校 / 年级 / 班级 / 教师等为固定档位；学生若存在 **`level in ('red','orange')` 的 Alert** 则节点更大，否则更小。
- 边 **`value`**：组织关系连边使用固定常数（如 1、0.5、0.3），**非由原始数据拟合的权重公式**。
