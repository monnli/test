# M0 · 基础设施完成清单

本里程碑目标：**搭好仓库骨架，让前端、后端、AI 服务三条腿都能独立启动并互相握手**。

> 本项目刻意不依赖 Docker / Redis，目标是「一台已装好 Python + MySQL 的笔记本就能跑起来」。

## ✅ 已完成

### 1. 仓库结构
- 完整目录骨架（backend / frontend / ai_service / scripts / docs）
- `.gitignore`、`.env.example`、`README.md`

### 2. Flask 后端
- 应用工厂模式 (`create_app`)
- 扩展集中管理（SQLAlchemy / JWT / Marshmallow / SocketIO / CORS / Migrate）
- 多环境配置类（Development / Production / Testing）
- 健康检查接口：`GET /api/health`、`GET /api/health/deep`（探测 MySQL / 文件存储 / AI 服务）
- 统一响应格式工具
- 全局错误处理（404 / 405 / 500）
- loguru 日志（按天滚动 + 控制台）
- 文件存储抽象 `app/utils/storage.py`（`local` / `minio` 双后端，默认 local）
- pytest 单元测试通过

### 3. AI 推理服务（FastAPI）
- 服务骨架 + 健康检查 + 设备信息接口
- 流水线目录结构（M2 起填充）
- CORS 配置

### 4. Vue 3 前端
- Vite + TypeScript + Vue 3.4
- Element Plus 中文化 + 图标自动注册
- Pinia / Vue Router
- 自动按需引入（unplugin-auto-import + unplugin-vue-components）
- 路径别名 `@/*`
- Axios 封装（统一拦截、错误提示、Token 注入）

### 5. 文档
- README（产品介绍、技术栈、快速开始、里程碑）
- 架构设计.md
- 伦理设计说明.md
- M0-完成清单.md（本文档）

## 🧪 验证方式

```bash
# 1. MySQL 已在本机运行
# 2. 安装后端依赖
conda activate kt_cursor
cd backend && pip install -r requirements.txt && cd ..

# 3. 配置环境
cp .env.example .env

# 4. 启动后端
cd backend && python run.py
# 访问 http://localhost:5000/api/health/deep

# 5. 启动 AI 服务
cd ai_service && pip install -r requirements.txt && python server.py
# 访问 http://localhost:8000/health

# 6. 启动前端
cd frontend && npm install && npm run dev
# 访问 http://localhost:5173
```

### 后端单元测试

```bash
cd backend && pytest
```
