# 青苗守护者 · 架构设计

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      浏览器 (Vue 3 SPA)                          │
│  数据大屏 / 管理后台 / 课堂分析 / 心理档案 / AI 对话 / 量表       │
└────────────┬───────────────────────────────────┬────────────────┘
             │ HTTPS REST                        │ WebSocket
             ↓                                   ↓
┌─────────────────────────────────────────────────────────────────┐
│              Flask 后端 (gevent + Flask-SocketIO)                │
│  用户/权限 RBAC | 业务逻辑 Service | 异步任务 | WS 推送          │
└─────┬─────────────┬─────────────┬───────────────────────────────┘
      ↓             ↓             ↓
   MySQL      本地存储 / MinIO     AI 推理服务 (FastAPI)
   业务数据    视频/文件             ↓
                              本地模型 + 通义千问 API
```

> **本项目刻意不依赖 Docker / Redis**，目标是「一台已装好 Python + MySQL 的笔记本就能跑起来」，方便比赛演示与各种环境部署。

## 二、模块划分

### 后端（Flask）

- `app/api/`：REST 蓝图集合（认证、用户、组织架构、健康检查等）
- `app/models/`：SQLAlchemy ORM 模型
- `app/schemas/`：Marshmallow 序列化与校验
- `app/services/`：业务逻辑层
- `app/ai/`：AI 客户端封装（调用 ai_service 微服务、通义千问 SDK）
- `app/tasks/`：异步任务（基于 Python threading + 进程内任务表，无需 Celery/Redis）
- `app/sockets/`：WebSocket 事件
- `app/utils/`：通用工具，含 `storage.py` 文件存储抽象（`local` / `minio` 双后端）

### AI 推理服务（FastAPI）

- `pipelines/behavior_pipeline.py`：YOLOv8 + DeepSORT，行为检测与追踪
- `pipelines/emotion_pipeline.py`：HSEmotion，表情识别
- `pipelines/face_pipeline.py`：InsightFace，人脸检测与识别
- `pipelines/text_pipeline.py`：通义千问，文本情绪/风险分析
- `server.py`：FastAPI 入口

为什么独立成微服务：

1. AI 依赖（PyTorch / CUDA）很重，与业务后端解耦，避免拖累后端启动
2. 可独立水平扩展（多实例分发）
3. AI 服务用 GPU 镜像，业务后端用普通环境

### 前端（Vue 3）

- `views/dashboard/`：DataV 风格数据大屏
- `views/workbench/`：工作台首页
- `views/profile/`：个人中心、修改密码、数据可见范围展示
- `views/system/`：用户、角色、权限管理
- `views/org/`：学校 / 年级 / 班级 / 学生 / 教师 / 科目 / 任课关系
- `views/classroom/`：课堂视频上传与分析、实时摄像头
- `views/psychology/`：心理量表、文本分析、AI 对话
- `views/correlation/`：多维关联分析
- `views/alerts/`：预警中心、工单流转
- `views/reports/`：报告查看与导出
- `layouts/MainLayout.vue`：可折叠侧边栏 + 顶栏 + 面包屑
- `stores/`：Pinia 状态管理（用户、权限）

## 三、数据流

### 1. 上传视频分析流程

```
用户上传视频 → Flask → 本地存储 / MinIO → 创建分析任务 (DB)
   ↓
后端线程池拉取任务 → 调用 AI 服务 (HTTP) → 接收分析结果
   ↓
写入 DB (behavior_records / emotion_records) → 通过 WS 推送进度给前端
   ↓
前端展示分析报告（视频回放 + 时间轴 + 多维图表）
```

### 2. 实时摄像头分析流程

```
浏览器 WebRTC 采集 → 抽帧 → WebSocket 推送 base64 帧
   ↓
Flask SocketIO 接收 → 转发到 AI 服务 → 推理
   ↓
推理结果通过 WebSocket 实时推回前端 → Canvas 叠加渲染
```

### 3. 心理量表流程

```
教师选择学生 → 学生作答 → 后端自动评分 → 写入 scale_assessments
   ↓
若评分超过阈值 → 创建预警工单 → 站内消息通知心理老师
```

### 4. 关联分析流程

```
后端定时任务 → 拉取学生多维数据 (成绩 + 行为 + 表情 + 量表)
   ↓
计算相关系数 / 异常评分 / 综合风险等级
   ↓
更新预警状态 → 大屏 / Dashboard 实时刷新
```

## 四、权限模型

| 角色 | 数据范围 | 特殊权限 |
|---|---|---|
| 学校管理员 | 本校全部 | 用户管理、配置 |
| 心理学老师 | 本校全部 | 同管理员 + 预警工单处理 |
| 年级组长 | 本年级 | 年级报告 |
| 班主任 | 本班全部科目 + 自任教科目跨班 | 班级报告 |
| 科任老师 | 自任教的「班级×科目」交叉范围 | 课堂分析 |

实现方式：基于（学校、年级、班级、科目）四元组做查询过滤；权限规则在 `app/utils/permissions.py` 的 `compute_data_scope()` 统一计算。

## 五、文件存储设计

```python
from app.utils.storage import get_storage

storage = get_storage()
storage.save("videos/abc.mp4", file_bytes)
url = storage.get_url("videos/abc.mp4")
```

后端通过 `STORAGE_BACKEND` 环境变量切换：

- `local`（默认）：写入 `LOCAL_STORAGE_DIR`，无任何外部依赖
- `minio`：使用 MinIO 对象存储（需自行启动 MinIO）

## 六、部署拓扑

### 开发机（GPU P100）

```bash
# 一台机器同时跑前端、后端、AI 服务
python scripts/init_db.py
python scripts/seed_demo_data.py
cd backend && python run.py &
cd ai_service && python server.py &
cd frontend && npm run dev
```

### 演示机（无 GPU）

```bash
AI_DEVICE=cpu          # 强制 CPU 推理
# 模型用 ONNX CPU 版本
# 部分演示数据预生成，避免现场卡顿
```
