# 青苗守护者 · 作品应用场景与设计理念

> 面向「全国大学生计算机设计大赛」作品文档。本文说明：作品解决什么问题、适用于哪些场景、为什么这样设计。

---

## 一、作品应用场景

### 1. 校园日常管理场景（学校管理层/心理中心）

- **场景**：校长/德育处/心理中心希望以较低成本掌握全校心理健康态势，及时发现需要关注的群体与风险趋势。
- **痛点**：
  - 传统量表测评频次低、滞后，难以及时发现状态波动
  - 数据割裂（课堂、量表、成绩、文本表达分散在不同系统/表格）
  - 预警缺乏闭环：发现问题后“谁处理、怎么处理、有没有效果”难追踪
- **本作品提供**：
  - 数据大屏（全校态势与风险流）
  - 四级风险预警 + 工单流转 + 干预记录 + 效果追踪
  - 可解释归因：每条预警附带原因依据，降低“黑盒恐惧”

### 2. 班级管理场景（班主任/年级组长）

- **场景**：班主任需要在“有限时间内”识别班级中需要额外关注的学生；年级组长关注年级层面整体趋势与重点班级。
- **痛点**：
  - 班主任对学生的观察高度依赖经验，易遗漏“沉默型”风险
  - 年级组长难以从多个班级中快速定位“异常班级/异常时段”
- **本作品提供**：
  - 班级/年级范围内的可视化与预警列表（按权限收敛数据范围）
  - 课堂行为与参与度指标，辅助判断“课堂参与—心理状态”的关联

### 3. 课堂辅助场景（科任老师/课堂分析）

- **场景**：科任老师希望掌握自己任课班级的课堂参与度、异常行为（如玩手机、趴桌等）趋势，用于改进课堂管理与教学策略。
- **痛点**：
  - 课堂状态难以量化，教师只能“感觉”
  - 实时与课后反馈断裂，难以复盘
- **本作品提供**：
  - 课表驱动的课堂识别（可自动化）
  - 摄像头墙与实时直播页（实时叠加分析结果）
  - 课表冲突检测、按权限过滤“班级×科目”可见性

## 二、设计理念（核心原则）

### 1. 从“监控”转向“守护”

- 作品强调**预警优于评价**：发现需要帮助的孩子，而非打分排名。
- 作品强调**避免标签化**：不输出永久性标签，仅输出“近期需要关注”的状态与证据。

### 2. AI 做“副驾驶”，人类做“主驾驶”

- AI 仅做辅助筛查与提示；高风险情形必须走**人工介入闭环**。
- 用工单与记录把“干预”变成可追踪流程，避免风险被忽略。

### 3. 可解释优先：让老师敢用、愿用

- 每条预警附原因清单与证据来源（多模态交叉验证），降低黑盒风险。
- 以“趋势/异常点”呈现数据，而非单次孤立结论。

### 4. 低门槛落地：任何机器都能演示与部署

- 系统刻意不强依赖复杂基础设施，支持**Mock 降级**，演示机无 GPU 也能跑通流程。
- 通过一键初始化与一键启动脚本，保证作品可复现、可验收。

---

## 三、目标用户与价值

- **学校管理者**：快速掌握全校态势，提升心理健康治理效率
- **心理老师**：减少重复性筛查工作，把时间放在“真正需要的孩子”
- **年级组长/班主任**：在可控范围内更早识别风险，形成家校沟通依据
- **科任老师**：掌握课堂参与度与异常行为趋势，优化教学管理

## 四、边界与声明（比赛文档建议必备）

1. 本作品所有分析结论仅供辅助决策参考，不构成医学诊断。
2. 真实部署需遵循学校/家长知情同意、数据合规与伦理审查流程。
3. 作品遵循“负责任 AI”原则，详见 `docs/伦理设计说明.md`。
