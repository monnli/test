## 一、源代码组成

本项目源代码位于仓库根目录 `qingmiao-guardian/`，主要包含三部分：

1. **前端**（Vue 3）：`frontend/`
2. **业务后端**（Flask）：`backend/`
3. **AI 推理服务**（FastAPI）：`ai_service/`

辅助目录：

- `scripts/`：一键初始化、数据 seed、模型下载、压测脚本
- `docs/`：比赛文档（方案、架构、伦理、演示脚本、Q&A）
- **`算法.md`**（仓库根目录）：各模块指标与评分公式说明，与实现对齐
- `storage/`：本地文件存储目录（运行后自动创建）

---

## 二、运行与验收（推荐方式）

### 方式 A：Windows 一键运行（最适合比赛验收）

1. 复制 `.env.example` 为 `.env`（默认即可）
2. 执行初始化：

```bat
scripts\init_demo.bat
```

3. 启动所有服务：

```bat
scripts\start_all.bat
```

4. 打开浏览器：`http://localhost:5173`

> 默认账号：`admin / admin123`（更多账号见 `docs/M1-完成清单.md`）

### 方式 B：Docker 一键启动（演示机无开发环境）

参见 `docs/Docker部署指南.md`。

## 三、依赖与环境说明

### 1) Python 版本

- 推荐：Python 3.10+

### 2) 依赖文件

- 后端依赖：`backend/requirements.txt`
- AI 服务依赖：`ai_service/requirements.txt`
- 根目录聚合安装（可选）：`requirements.txt`

> 说明：AI 服务的“真实模型依赖”默认注释，可按 `docs/CPU降级部署说明.md` 与 `docs/真实模型激活说明.md` 启用。

### 3) 数据库

- MySQL 8.0（本机安装即可）
- 参数在 `.env` 中配置（默认开发配置可直接跑 demo）

## 五、可复现性说明（比赛非常关键）

本项目提供“一键初始化 + 一键启动”能力，确保评委在不同电脑上可快速复现：

- 初始化脚本：`scripts/init_demo.*`
  - 建表、写入演示学校/班级/学生/教师/账号
  - 写入增强演示数据（量表/文本/预警/报告等）
  - 生成演示视频、写入摄像头与课表（M10）
- 启动脚本：`scripts/start_all.*`
  - 一次启动前端、后端、AI 服务
- Mock 降级：AI 模型不可用时自动降级，不影响演示链路（见 `docs/CPU降级部署说明.md`）
