# 青苗守护者 · 技术方案（含架构、模块、关键流程）

> 面向「全国大学生计算机设计大赛」作品技术文档。本文强调“方案完整可落地”，并与代码实现一一对应。

---

## 一、总体架构

系统为三服务结构：

- **前端**：Vue 3 SPA（管理后台 + 数据大屏）
- **业务后端**：Flask（REST + WebSocket + 异步任务）
- **AI 推理服务**：FastAPI（多条流水线，可真可假）

数据与文件：

- **MySQL**：业务数据
- **本地磁盘 / MinIO**：视频、人脸库、报告文件存储（可切换）

> 架构图与数据流详见 `docs/架构设计.md`。

## 二、核心模块设计

### 1) 用户与权限（RBAC + 数据范围）

- 角色分级（5 级 + 隐藏超管）：
  - 学校管理员/心理老师：本校全部
  - 年级组长：本年级
  - 班主任：本班全科 + 自任教科目跨班（班级×科目）
  - 科任老师：仅任课的班级×科目
- 数据范围统一计算：`backend/app/utils/permissions.py::compute_data_scope()`
- 路由/菜单按角色裁剪：前端 `stores/user.ts` + `router/index.ts` + `layouts/MainLayout.vue`

### 2) 组织架构（学校/年级/班级/学生/教师/任课关系）

- 后端接口：`backend/app/api/orgs.py`
- 业务逻辑：`backend/app/services/org_service.py`
- 前端页面：`frontend/src/views/org/*`

> 权限边界：学校管理（新建/编辑/删除学校）仅超级管理员；学校管理员只管理本校数据。

### 3) 课堂分析（离线视频库、摄像头、课表、实时直播）

**后端接口**：

- **离线视频**：`backend/app/api/classroom.py` — 视频 CRUD、分析任务、任务报告、**单帧检测** `POST .../videos/<id>/detect-frame`（报告页回放用）
- **M10 摄像头/课表/实时**：`backend/app/api/cameras.py`（`/m10/cameras`、`/m10/schedules`）、`backend/app/api/live.py`（`/m10/live/*`）

**关键特性**：

- 离线视频：上传 → 抽帧 → 调 AI 行为/情绪 → 写入 `behavior_records` / `emotion_records`；行为标签在展示层 **归一为八类中文**（`video_service._normalize_behavior_cn`）
- 课表 CRUD + 冲突检测（班级/教师/时段）
- 课表驱动调度器：按时间启动/停止课堂识别（详见 `docs/M10-视频识别深度改造完成清单.md`）
- 数据范围过滤：科任老师仅可见其任课的“班级×科目”课程；班主任本班可见全科

**前端入口**：

- 视频库：`/classroom/videos`
- 上传视频：`/classroom/upload`
- 摄像头墙：`/classroom/cameras`
- 摄像头管理（管理员）：`/classroom/camera-manage`
- 课表管理：`/classroom/schedule`
- 实时直播：`/classroom/live/:cameraId`

### 4) 心理健康（量表、档案、对话、文本）

**目标**：用标准量表 + 文本表达 + 对话记录作为心理风险的“多证据”来源。

- 量表：PHQ-9 / GAD-7 / SCARED / CES-DC / MHT
- 结果沉淀到学生档案，并可触发预警
- 高风险对话提示热线并触发通知/工单（详见 `docs/答辩问答手卡.md` 与 `docs/伦理设计说明.md`）

前端入口：`/psychology`

### 5) 关联分析与预警闭环

**关联分析**：

- 目标：把心理、课堂、学业等多源数据“放到同一张图里”
- 前端入口：`/correlation`

**预警闭环**：

- 4 级风险（绿/黄/橙/红）
- 工单流转、干预记录、前后对比
- 前端入口：`/alerts` 与 `/enhance/intervention`

### 6) 数据大屏与报告中心

- 数据大屏：`/dashboard`（面向校级管理者的“态势”页面）
- 报告中心：`/reports`（在线查看 + PDF 下载）

## 三、关键流程（从数据到闭环）

### 1) 视频/课堂分析流程（课表驱动）

1. 初始化：`scripts/seed_cameras_and_schedules.py` 写入摄像头与学期课表
2. 调度：后端周期扫描活跃课程 → 自动创建 session → 启动分析
3. 推理：后端调用 AI 服务流水线 → 写入结果 → WebSocket 推送给前端
4. 展示：摄像头墙/实时直播页实时叠加；异常可触发预警

### 2) 心理测评与预警流程

1. 老师下发/组织测评 → 学生作答
2. 后端自动评分分级 → 写入档案
3. 触发阈值 → 创建预警工单 → 心理老师签收处理 → 记录干预
