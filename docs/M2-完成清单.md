# M2 · AI 推理服务完成清单

本里程碑目标：**搭建一套「可真可假」的 AI 推理能力**——装了模型用真实算法，不装模型或机器没 GPU 也能以 mock 模式跑通完整链路，保证任何开发/演示环境都能用。

## ✨ 设计亮点

1. **流水线基类 + 延迟加载 + 自动降级**：所有 AI 流水线继承 `BasePipeline`，模型加载失败自动切换到 mock，业务不中断
2. **mock 模式的价值**：
   - 前后端团队可以在没有 GPU 的机器上开发联调
   - 单元测试不依赖模型权重（12 个 AI 测试在 0.07 秒跑完）
   - 演示机容灾：真实模型加载失败，系统降级而非崩溃
3. **基于图像 hash 的稳定 mock**：同一张图每次得到同一个结果，方便 UI 调试

## ✅ 已完成

### 1. AI 推理服务（FastAPI）

#### 流水线（4 条）

| 流水线 | 真实模型 | mock 行为 |
|---|---|---|
| **face** | InsightFace buffalo_l（512 维） | 基于图像哈希生成伪 embedding，保持同图同向量 |
| **emotion** | HSEmotion ENet-B0 | 7 类情绪 Dirichlet 随机分布 |
| **behavior** | YOLOv8n（COCO person/cell phone/book 等） | 随机生成 2~5 个 person 框 |
| **text** | 通义千问 qwen-plus API | 基于关键词词典的情绪极性 + 4 段预设对话模板 |

#### REST 接口

- `GET  /health` `/info` `/pipelines`
- `POST /pipelines/{name}/load`
- `POST /face/detect`（base64 图 → 检测 + embedding）
- `POST /face/match`（1:N 比对，带阈值）
- `POST /emotion/predict`
- `POST /behavior/detect`
- `POST /text/sentiment`（情绪极性 + 风险等级 + 情绪标签）
- `POST /text/summarize`（心理状态归纳）
- `POST /text/chat`（带系统提示词的心理对话，含风险词检测）

#### 安全设计

- 风险关键词词典（12 个 high + 13 个 medium + 10 个 low）
- `text/chat` 检测到高风险自动返回「请联系老师或拨打 12320」
- LLM 系统提示词明确「不做诊断、遇到自伤立即标记」

### 2. 后端

#### 数据模型

- 新增 `face_embeddings` 表（student_id + 512 维 embedding JSON + 图片 hash 去重 + 原图 URL）

#### AI 客户端封装（`app/ai/client.py`）

- 统一 `AIClient`（httpx），封装所有 AI 服务调用
- 失败优雅降级（返回 code=-1 + 空 data，不抛异常）
- 7 个业务方法：`face_detect` / `face_match` / `emotion_predict` / `behavior_detect` / `text_sentiment` / `text_summarize` / `text_chat`

#### REST 接口

- `/api/faces/*`：人脸库管理（注册/识别/删除/统计）
  - `GET  /api/faces/stats` 全库统计
  - `GET  /api/faces/by-student/{id}` 查某学生人脸
  - `POST /api/faces/by-student/{id}` 注册（支持 multipart 和 JSON/base64 两种方式，自动去重）
  - `DELETE /api/faces/{id}` 删除单张
  - `DELETE /api/faces/by-student/{id}` 删除某学生全部人脸
  - `POST /api/faces/recognize` 人脸识别（在当前用户可见的学生范围内）
- `/api/ai/*`：AI 服务调试（健康、文本分析、图像分析）

#### 数据范围权限

- 注册人脸：`@admin_required`
- 识别人脸：默认在当前用户数据范围内搜索（班主任只能识别本班学生，科任老师只能识别任教班级学生）
- 查询/删除：需能访问目标学生

### 3. 前端

#### AI 服务监控页（`/system/ai`）

- 实时健康状态卡片 + 推理设备展示
- 4 条流水线状态表（就绪/降级/未加载）+ 手动加载按钮
- 3 个在线测试工具：
  - 文本情绪分析（极性 + 风险等级 + 情绪标签 + 依据）

- 4 张统计卡片（人脸总数 / 覆盖学生 / 学生总数 / 覆盖率）
- 左侧：学生列表（带班级过滤 + 关键词搜索 + 已注册张数角标）
- 右侧上方：选中学生的人脸缩略图网格 + 上传注册按钮
- 右侧下方：人脸识别测试（上传图片 → 匹配到哪位学生）
