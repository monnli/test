# 青苗守护者 · 用户手册

> 面向比赛验收与真实使用的操作手册：包含登录、角色使用、核心功能流程、常见问题。

---

## 一、快速开始

### 1. 初始化与启动

Windows（推荐）：

1. 初始化演示数据：

```bat
scripts\init_demo.bat
```

2. 启动全部服务：

```bat
scripts\start_all.bat
```

3. 浏览器访问：`http://localhost:5173`

Docker 一键启动：见 `docs/Docker部署指南.md`

登录页提供“演示账号一键填充”。常用账号如下（完整列表见 `docs/M1-完成清单.md`）：

## 三、角色与数据范围（你为什么看不到别人的数据）

系统为 5 级精细权限，所有列表与图表会按权限自动过滤：

- 学校管理员/心理老师：本校全部数据
- 年级组长：仅本年级
- 班主任：本班全科 + 自任教科目跨班
- 科任老师：仅自任教的班级×科目

> 提示：学校管理（新建/编辑/删除学校）仅超级管理员可用；学校管理员默认只管理本校。

## 四、功能使用指南（按菜单顺序）

### 1) 工作台

入口：`工作台`

- 查看当天提醒、快捷入口与系统概览
- 建议作为讲解起点（比赛演示更顺）

### 2) 数据大屏

入口：`数据大屏`

- 查看全校/本校态势（按角色范围显示）
- 适合开场“先讲价值，再讲功能”

### 3) 组织架构

入口：`组织架构`

- 年级管理：新增/编辑年级（管理员）
- 班级管理：新增/编辑班级（管理员）
- 学生管理：维护学生档案
- 教师管理、科目管理、任课关系：用于配置“谁教哪个班的哪门课”（管理员）

> 注意：学校管理员账号下“学校管理”入口默认隐藏；科任老师/班主任/年级组长可见的数据会更少，这是权限设计。

### 4) 课堂分析

#### 4.1 摄像头墙

入口：`课堂分析 → 摄像头墙`

- 查看各教室摄像头状态与实时态势
- 点击卡片进入实时直播页

#### 4.2 实时直播

入口：`课堂分析 → 实时直播`

- 查看实时检测结果叠加（行为/情绪/身份标签等）
- 适合现场演示“实时能力”

#### 4.3 课表管理

入口：`课堂分析 → 课表管理`

- 查看课表（周视图/列表视图）
- 管理员可新增/编辑课程，并进行冲突检测
- 科任老师仅可看到自己任课科目；若同时为班主任，本班可见全科

#### 4.4 视频库与上传

入口：`课堂分析 → 视频库`、`课堂分析 → 上传视频`

- **视频库**：按权限查看已上传视频列表，进入**任务详情**查看异步分析进度与报告。
- **上传视频**：选择文件、班级（可选）、抽帧间隔后上传；若存在未完成任务会提示沿用原任务。
- **删除**：在视频库中对有权限的视频执行删除（将移除存储文件及关联分析任务与检测记录，请谨慎操作）。

#### 4.5 任务详情与课堂报告

入口：从视频库进入某视频 → 选择或进入分析任务

- **进度与摘要**：异步分析完成后可查看行为/情绪汇总与时间轴（行为按 **标准八类** 聚合展示）。
- **视频回放**：播放原片；可对**当前帧**请求实时行为检测（YOLO 画框），用于答辩演示「单帧可解释」。
- **指标说明**：大屏与报告中的参与度等公式见仓库根目录 **`算法.md`**。

### 5) 心理健康

入口：`心理健康`

- 量表测评：选择量表、下发/组织测评、自动评分分级
- 学生档案：查看学生长期趋势、历史测评、文本分析与对话记录

### 6) 关联分析

入口：`关联分析`

- 从多源数据角度观察心理与学业/课堂的关联关系
- 用于解释“为什么会触发预警”

### 7) 预警中心

入口：`预警中心`

- 查看四级预警列表
- 点击查看原因清单与证据来源
- 记录干预动作，形成可追溯闭环

### 8) 报告中心

入口：`报告中心`

- 查看报告内容（在线）
- 下载 PDF（用于归档与汇报）

### 9) 负责任 AI

入口：`负责任 AI`

- 查看系统在隐私、边界、可解释等方面的设计承诺
- 建议答辩时作为加分项重点讲

## 六、常见问题（FAQ）

### Q1：为什么我看不到某些班级/科目/学生？
这是权限设计。系统按角色与任课关系自动过滤数据范围，确保“只看该看的”。

### Q2：演示机没有 GPU、AI 模型跑不动怎么办？
按 `docs/CPU降级部署说明.md` 设置 CPU 模式，或启用 Mock 降级，流程仍可完整演示。

### Q3：我想重置演示数据
重新运行 `scripts/init_demo.bat`（会重建并写入演示数据；若数据库配置变更请先更新 `.env`）。
