# M1 · 用户与权限完成清单

本里程碑目标：**完整可用的 5 级 RBAC 权限体系 + 组织架构 CRUD + 一份可演示的种子数据**。

## ✅ 已完成

### 1. 数据模型（14 张核心表）

| 表 | 说明 |
|---|---|
| `schools` | 学校 |
| `grades` | 年级 |
| `classes` | 班级（含班主任 ID） |
| `students` | 学生 |

| `user_roles` | 用户-角色绑定 |
| `operation_logs` | 操作日志（M1 内仅建表） |

### 2. 认证与权限

- `bcrypt` 密码哈希
- JWT 登录 / 刷新 / 登出 / 当前用户接口
- 5 级角色 + 1 隐藏超管：
  - `super_admin` 平台超管（全部数据）
  - `school_admin` 学校管理员（本校全部）
  - `psy_teacher` 心理学老师（同学校管理员）
  - `grade_head` 年级组长（所任年级）
  - `head_teacher` 班主任（本班全部科目 + 自任教科目跨班）
  - `subject_teacher` 科任老师（自任教的「班级×科目」交叉范围）
- `DataScope` 数据范围模型 + `compute_data_scope()` 计算函数
- 权限装饰器：`@login_required` / `@roles_required(...)` / `@admin_required`
- 23 个权限点 + 6 个内置角色 + 角色-权限映射

### 3. REST API（约 40 个接口）

- `/api/auth/*`：登录、刷新、登出、当前用户、修改密码
- `/api/users/*`：用户增删改查、重置密码、角色分配、查角色列表
- `/api/orgs/schools|grades|classes|students|teachers|subjects/*`：组织架构 CRUD
- `/api/orgs/teacher-class-subjects/*`：任课关系增删改查
- `/api/orgs/grade-heads/*`：年级组长任命/撤销
- 所有 list 接口都带数据范围过滤；所有写接口都带角色权限拦截

### 5. 前端

- 登录页：演示账号一键填充、登录后跳转

- 教师管理 / 科目管理 / 任课关系管理

## 🧪 验证步骤

```bash
# 1. 启动 MySQL/Redis（任选）
docker compose up -d mysql redis

# 2. 配置 .env
cp .env.example .env

# 3. 安装后端依赖
conda activate kt_cursor
cd backend && pip install -r requirements.txt

# 4. 初始化数据库 + 写入演示数据
cd ..
python scripts/init_db.py
python scripts/seed_demo_data.py

# 5. 启动后端
cd backend && python run.py

# 6. 启动前端（另一个终端）
cd frontend && npm install && npm run dev

# 7. 浏览器打开 http://localhost:5173
#    点击演示账号一键填充 → 登录 → 体验各角色权限差异
```

## 🎯 体验建议（评审顺序）

1. 用 `admin` 登录 → 体验全校管理（组织架构所有页面）
2. 切换 `head_7_1b`（七1班班主任）→ 学生只能看到七1班 30 人
3. 切换 `sub_physics`（物理老师）→ 学生能看到全部 180 人（教全校物理）

6. 进入「个人中心」→ 查看「数据可见范围」可视化展示
