# 青苗守护者 · 负责任 AI 设计说明

> 在中小学场景中使用 AI 进行行为分析与心理健康评估，必须将「伦理」作为系统的一等公民，而非事后补丁。本文档说明本平台的伦理设计原则与对应的技术实现。

## 一、核心原则

1. **辅助而非替代**：AI 仅做辅助筛查与预警，不替代专业心理咨询师/医生的最终判断。
2. **隐私最小化**：仅采集与目标直接相关的最小必要数据，避免「能采尽采」。
3. **知情与可控**：使用者（学校）与被分析对象（学生）应清楚知道哪些数据被采集、用于何种目的、如何申诉与删除。
4. **避免标签化**：系统不输出「抑郁学生」「问题学生」等永久性标签，只输出「需要关注的状态」。
5. **预警优于评价**：系统目标是「发现需要帮助的孩子」，不是「评比/排名/筛选学生」。
6. **人工介入闭环**：所有红色预警必须由真人专业人员处理，AI 不做最终决定。

## 二、技术落实

| 维度 | 落实方式 |
|---|---|
| 数据采集 | 仅采集课堂场景音视频；非课堂场景不采集 |
| 人脸数据 | 提取后立即转换为 512 维特征向量；原图加密存储或丢弃；可哈希匿名化 |
| 心理量表 | 使用国内/国际公开标准化量表，并在结果页注明「本结果仅供筛查参考，不构成诊断」 |
| AI 对话 | 系统提示词中明确「我不是医生，遇到自伤/自杀想法请立即联系老师/12320 热线」 |
| 风险评分 | 多维证据加权 + 置信度，避免单一维度误判 |
| 红色预警 | 强制工单流转 → 心理老师签收 → 干预记录 → 跟踪关闭 |
| 数据访问 | RBAC 严格分级；普通教师看不到敏感心理细节，仅看「绿/黄/橙/红」状态 |
| 操作审计 | 所有敏感数据访问留痕 |
| 数据生命周期 | 学生离校后默认软删除；可应申请彻底删除 |

## 三、对教师与管理者的提示

- 系统输出仅是参考信息之一；请结合家访、谈话、家长沟通等综合判断
- 看到红色预警，不是「这个学生有病」，而是「这个学生最近需要更多关注」
- 不要在公开场合宣读个体心理评分；不要在班级排名中混入心理数据

## 四、对评委与使用方的承诺

- 全部 AI 分析支持本地化部署，敏感数据可不出校园
- 算法可解释：每条预警附「原因依据」，不做黑盒
- 关键阈值可配置，由学校心理学教师调整
- 开源核心代码，接受社会监督
