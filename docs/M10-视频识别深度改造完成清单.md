# M10 · 视频识别深度改造完成清单

## ✅ 已完成 · 5 大子里程碑

### V1 · 数据模型 + 课表管理
- 2 张新表：`cameras` / `class_schedules`
- `class_sessions` 扩展 6 个字段（camera_id / trigger_type / status / started_at / ended_at / engagement_score）
- 摄像头 CRUD（绑定班级 + 5 种流类型 + 在线状态 + 心跳）
- 课表 CRUD + **冲突检测**（班级冲突 + 教师冲突 + 时段重叠）
- `find_active_schedules` 查当前正在进行的课
- API 全部在 `/api/m10/*`

### V2 · AI 流水线深度升级
- **PosePipeline**：MediaPipe Pose 33 关键点 + FaceMesh 头部 pitch
- **SimpleTracker**：零依赖 IoU 追踪器（替代 DeepSORT）
- **ClassroomPipeline**：综合分析 = YOLO + Tracker + InsightFace + Pose + Emotion 联动
- **9 种行为派生**：举手 / 趴桌 / 坐姿 / 站立 / 睡觉 / 抬头 / 低头 / 玩手机 / 交头接耳
- **参与度评分公式**：抬头×40 + 举手×20 + 正向情绪×20 + (1-趴桌)×10 + (1-玩手机)×10
- AI 服务新接口：`POST /classroom/analyze`、`POST /classroom/reset-tracker/{key}`
- 测试：13 个 AI 测试全过

### V3 · 实时摄像头系统
- **LiveWorker**：OpenCV 拉流 + AI 综合分析 + WebSocket 实时推送
  - 支持 `rtsp / hls / file / file_loop / webrtc` 5 种流
  - 3 秒抽一帧，每 N 帧做一次人脸识别
  - **5 分钟空教室自动休眠**
  - 出勤自动写 `attendance_records`
- **Scheduler**：课表驱动调度器（每 60 秒扫描）
  - 找活跃课 → 自动创建 session → 启动分析
  - 课程结束时间到 → 自动停止
- **合成演示视频**：`generate_demo_videos.py`
  - 5 段 30 秒 1280×720 mp4（模拟 5 个班级）
  - 6×5 座位布局、30 个虚拟学生
  - 随机举手 / 趴桌动画
  - 中文字幕（时间、班级、科目）
- **学期课表 seed**：2026 春季（02-17 ~ 07-05）· 每班每周 25 节课

### V4 · 前端大改造
- **`/classroom/cameras`** 摄像头墙：网格缩略图 + LIVE 标识 + 一键启停
- **`/classroom/live/:cameraId`** 实时直播：
  - 视频画布 + Canvas 叠加检测框（绿/橙/红 3 色）
  - 实时人数、参与度（巨型数字 + 动态颜色）
  - 5 种行为实时进度条
  - WebSocket 接收 `live_analysis` 事件
  - 已识别学生标签云
- **`/classroom/camera-manage`** 摄像头管理：5 种流类型 CRUD
- **`/classroom/schedule`** 课表管理：
  - 周视图（班级筛选自动切换）+ 列表视图
  - 实时冲突检测（新建时输入即校验）
  - 学期起止日期范围

### V5 · 真实测试 + 模型下载 + 兜底
- `scripts/benchmark_video.py` 压测脚本：
  - 对任意课堂视频跑完整流水线
  - 输出 HTML 报告（指标卡 + 行为/情绪分布柱图 + 参与度曲线 + 6 张关键帧截图）
  - 支持答辩 PPT 直接截图
- `scripts/download_models.py` 新增 MediaPipe 检查
- `scripts/download_test_videos.py` + README：指导如何合规获取真实课堂视频
- `init_demo.sh/.bat` 扩展为 5 步：DB 建表 → 基础 seed → 增强 seed → 视频合成 → 摄像头课表

## 🎯 答辩演示建议

### 路径 1：课表驱动全自动（高难度，需网络稳定）
1. 进入 `/classroom/schedule` 临时新建一节"当前时间"的课
2. 等 60 秒自动触发调度器
3. 切到 `/classroom/cameras` 看到对应摄像头变红色 LIVE
4. 点进 `/classroom/live/X` 看实时检测画面
5. 说明："整个流程从老师走进教室到 AI 开始工作，无需任何人工操作"

### 路径 2：摄像头墙手动启停（稳妥）
1. `/classroom/cameras` 点击任一摄像头"开启分析"
2. 等 3 秒进入 LIVE 状态
3. 点进实时直播页看叠加效果
4. 说明："支持 RTSP、HLS、本地文件多种协议"

### 路径 3：上传真实视频压测报告（展示技术深度）
```bash
python scripts/benchmark_video.py path/to/real_classroom.mp4
open benchmark_report.html
```
把 HTML 报告打印成 PDF 附在项目报告里。
